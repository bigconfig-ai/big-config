# Goal

Update the BigConfig `package` template so generated projects are a simplified,
compute-only version of `/Users/amiorin/code/personal/once/main/`, while keeping
the existing BigConfig package-template style.

This plan is for implementation work only; do not add tests for this change.

# Decisions

- Edit implementation files later; this plan update is documentation/planning only.
- Keep the generated workflow stages:
  1. `tofu` — provision or describe compute
  2. `ansible-local` — ping only
  3. `ansible` — ping only
- Keep `tofu-backend` support for `s3`, `r2`, and `local` remote state.
- Keep compute providers: `hcloud`, `oci`, `digitalocean`, and `no-infra`.
- Remove all DNS and SMTP concepts from the generated template.
- Keep `no-infra` support with IP/user/sudoer params.
- Use hardcoded placeholder values for SSH keys and provider settings in the
  generated `options.clj.source`; users edit the generated profile or override
  with `BC_PAR_*`.
- `describe` should report selected compute provider, compute IP, and SSH
  reachability only.
- `validation` should check required CLIs, cloud credentials, and that the
  configured compute SSH public key is loaded in `ssh-agent` for cloud compute
  profiles.

- `:owner`

Implementation notes:

- Remove `:ssh-key` from the `package` docstring and examples.

- Update the `package` transform list so it renders the new compute-only

- Preserve the existing package-template style for root/env/clj-kondo/lsp files.

## 2. Replace package source templates

Inside `resources/big-config/package/src/`, the template should contain these
six files and no extra Clojure source templates:

- `describe.clj.source`
- `options.clj.source`
- `package.clj.source`
- `params.clj.source`
- `tools.clj.source`
- `validation.clj.source`

Base these files on the existing BigConfig package template style, simplified
with the relevant ideas from the `once` project:

### `options.clj.source`

- Namespace uses `<{ namespace }>.options`.
- Define private profile fragments for:
  - backend: `s3`, `r2`, `local`
  - compute: `oci`, `hcloud`, `digitalocean`, `no-infra-compute`
  - deploy/SSH: at least `:compute-pubkey`
- Use hardcoded placeholder values for provider credentials/settings and SSH
  public keys.
- Provide public profiles for each compute provider, e.g. `oci-profile`,
  `hcloud-profile`, `digitalocean-profile`, and `no-infra`.
- Define `(def bb ...)` as the active profile.
- Do not include DNS, SMTP, domains, ONCE applications, or deploy-user app
  configuration.

### `params.clj.source`

- Namespace uses `<{ namespace }>.params`.
- Read `BC_PAR_*` overrides with `workflow/read-bc-pars`.
- Read compute params from `tofu output --json` for the generated tofu tool.
- Fallback to a placeholder IP like `192.168.0.1` when output is missing.
- Provide an opts function used by package/tools/tasks.
- Do not include SMTP/DNS output extraction.

### `tools.clj.source`

- Namespace uses `<{ namespace }>.tools`.
- Keep the `tofu-backend` render plugin pattern from `once`:
  - `pluggable/handle-step` for a backend render step after `render`
  - provider selected by `:provider-backend`
  - supports `s3`, `r2`, `local`
- Keep only these tool entry points:
  - `tofu` / `tofu*`
  - `ansible-local` / `ansible-local*`
  - `ansible` / `ansible*`
- `tofu` renders provider-specific compute templates selected by
  `:provider-compute` and supports `:compute-prevent-destroy`.
- `ansible-local` renders and runs only an Ansible ping playbook.
- `ansible` renders and runs only an Ansible ping playbook.
- Remove all `tofu-smtp`, `tofu-dns`, `tofu-smtp-post`, app deploy, and SMTP/DNS
  helper functions.

### `package.clj.source`

- Namespace uses `<{ namespace }>.package`.
- Create workflow pipeline:
  1. `::tools/tofu ["render tofu:init tofu:apply:-auto-approve" params/opts-fn]`
  2. `::tools/ansible-local ["render ansible-playbook:main.yml" params/opts-fn]`
  3. `::tools/ansible ["render ansible-playbook:main.yml" params/opts-fn]`
- Delete workflow pipeline destroys only `::tools/tofu`.
- Merge params into the relevant tool opts using `workflow/merge-params` or the
  existing package-template equivalent.
- Do not include SMTP/DNS stages.

### `describe.clj.source`

- Namespace uses `<{ namespace }>.describe`.
- Report:
  - selected compute provider
  - backend provider if useful
  - compute IP
  - SSH reachability (`ssh ... true` or equivalent)
- Exit non-zero only when the report command itself cannot complete in a clear,
  actionable way.
- Do not inspect ONCE applications, container images, registries, SMTP, or DNS.

### `validation.clj.source`

- Namespace uses `<{ namespace }>.validation`.
- Validate the active profile (`options/bb`).
- Check required CLIs:
  - common: `tofu`, `ansible-playbook`, `ssh`
  - provider-specific as needed (for example `hcloud`, `doctl`, `oci` if used by
    the credential checks)
- Check cloud credentials for `hcloud`, `oci`, and `digitalocean` profiles.
- Check that `:compute-pubkey` is loaded in `ssh-agent` for cloud compute
  profiles.
- Keep `no-infra` validation for required IP/user/sudoer params, but do not
  require cloud credentials or ssh-agent key for it.
- Remove SMTP, DNS, image, domain, and ONCE application validation.

## 3. Update resource templates

Under `resources/big-config/package/`, keep only compute-related tool templates:

- `tofu/oci/main.tf`
- `tofu/hcloud/main.tf`
- `tofu/digitalocean/main.tf`
- `tofu/no-infra/main.tf`
- `tofu-backend/s3/backend.tf`
- `tofu-backend/r2/backend.tf`
- `tofu-backend/local/backend.tf`
- `ansible/main.yml`
- `ansible/ansible.cfg`
- `ansible-local/main.yml`
- `ansible-local/ansible.cfg`
- `ansible-local/inventory.ini` if needed by the local ping playbook

Template requirements:

- Tofu templates should be simplified copies/adaptations of the `once` compute
  provider templates.
- Tofu outputs should include at least `ip`, `user`, `sudoer`, and `name` so
  `params`, `describe`, and Ansible can use them.
- Keep `lifecycle { prevent_destroy = <{ compute-prevent-destroy }> }` for cloud
  compute templates.
- `ansible/main.yml` should only ping the remote host.
- `ansible-local/main.yml` should only ping localhost/local inventory.
- Remove DNS, SMTP, ONCE app deployment, deploy user, mail, and container tasks.

Ensure these tasks use the generated `options/bb` profile and `params/opts-fn`.
